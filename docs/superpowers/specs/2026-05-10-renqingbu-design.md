# 人情簿 - 设计文档

## 概述

一个人情往来记录工具，手机网页形式，一家人共用。支持拍照OCR识别纸质本子的随礼记录，自动关联收礼与还礼，支持搜索和统计。

## 目标用户

一家人（多人共用），通过账号登录共享数据。

## 平台

手机浏览器网页（响应式设计，手机为主）。

---

## 技术方案

**Supabase 托管方案**：
- 前端：React + Tailwind CSS（单页网页应用）
- 后端服务：Supabase（提供登录认证、数据库、图片存储）
- OCR识别：百度云OCR接口（通用文字识别）
- 部署：Vercel / Netlify（免费静态站点托管）

---

## 页面结构（5个页面）

### 1. 登录页
- 邮箱注册/登录
- Supabase 自带认证组件

### 2. 首页/仪表盘
- 今年统计卡片：收礼总额、还礼总额、净支出
- "待还礼"提醒列表（别人随了礼但还没还的）
- 快速入口：手动添加、拍照添加

### 3. 记录列表页（总览）
- 按人汇总，每行显示：姓名、总收礼金额、总还礼金额
- 支持搜索（输入姓氏或名字实时筛选）
- 支持按事件类型筛选（红事/白事/生日/升学/乔迁/其他）
- 支持年份切换（今年/去年/全部）
- 支持排序（按日期、按金额高低）
- 点击某人进入详情页

### 4. 添加记录页
- 两种添加方式切换：手动填写 / 拍照识别
- 手动填写表单字段：姓名、关系、事件类型、事件描述、金额、日期、收支类型（收礼/还礼）、关联记录（还礼时可选）、备注
- 拍照识别流程：拍照 → 上传 → 百度OCR识别 → AI提取关键信息 → 用户确认 → 导入

### 5. 人员详情页
- 显示该人与我家的所有往来记录
- 每条记录显示：日期、事件类型、事件描述、金额、收支类型
- 收礼和还礼关联显示（哪笔还礼对应哪笔收礼）
- 汇总：总收、总还、差额

## 数据库设计

### 表：records（随礼记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| user_id | uuid | 所属用户（家庭共享通过同一账号） |
| person_name | text | 随礼人姓名 |
| relationship | text | 亲戚关系（大舅、邻居、同事等） |
| event_type | text | 事件类型：红事/白事/生日/升学/乔迁/其他 |
| event_desc | text | 事件描述（如"儿子结婚"） |
| amount | integer | 金额（元） |
| record_date | date | 日期 |
| direction | text | 收支类型：收礼(received) / 还礼(paid) |
| linked_record_id | uuid | 关联的对方记录（还礼关联到收礼） |
| status | text | 状态：待还礼(pending) / 已还(settled) / 无(仅还礼记录) |
| notes | text | 备注 |
| image_url | text | 拍照识别的原图地址 |
| created_at | timestamptz | 创建时间 |

## OCR识别流程

1. 用户在"添加记录"页点击"拍照添加"
2. 调用手机相机拍照（或从相册选择）
3. 图片上传到 Supabase Storage
4. 调用百度云通用文字识别API，获取图片中的文字
5. 前端用规则/AI从识别文字中提取：姓名、金额、事件、日期
6. 将提取结果显示在表单中，用户确认/修改
7. 确认后写入数据库

## 搜索与筛选

- **搜索**：输入文字实时过滤姓名
- **类型筛选**：红事/白事/生日/升学/乔迁/其他 标签切换
- **年份切换**：今年/去年/全部
- **排序**：按日期升序/降序、按金额升序/降序

---

## 统计看板（首页）

- 今年收礼总额
- 今年还礼总额
- 净支出（还礼 - 收礼）
- 待还礼人数和总额
- 待还礼列表（按日期排序，最近的在前）

---

## 非功能需求

- 手机优先设计，字体和按钮要大，方便手指操作
- 页面加载快（Supabase CDN + 静态站点）
- 数据安全（Supabase Row Level Security，同个家庭成员共享数据）
- OCR免费额度 1000次/月，个人使用足够
